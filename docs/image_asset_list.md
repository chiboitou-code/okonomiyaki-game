## 2. 調理フェーズ（ひっくり返しパート）

### 2-1. お好み焼き本体（表裏が入れ替わる5状態）
豚肉が乗った面（表）と生地の面（裏）が、ひっくり返すたびに交互に入れ替わる想定です。焼きが進むにつれて色味も変化させます。

| ファイル名 | 用途 | サイズ目安 | 透過 | 優先度 |
|---|---|---|---|---|
| `body_00_raw.png` | 焼く前（豚肉が乗った生の状態） | 800×800px | 要 | ★★★ |
| `body_01_backside.png` | 1回目ひっくり返した後（裏面、まだ薄い焼き色） | 800×800px | 要 | ★★★ |
| `body_02_porkside.png` | 2回目ひっくり返した後（豚肉面、焼き色が少し乗った状態） | 800×800px | 要 | ★★★ |
| `body_03_backside.png` | 3回目ひっくり返した後（裏面、さらに焼き色UP） | 800×800px | 要 | ★★★ |
| `body_04_porkside.png` | 4回目ひっくり返した後（豚肉面、完成状態＝トッピングフェーズのベースにも使用） | 800×800px | 要 | ★★★ |

| ~~`hotplate.png`~~ | ～~鉄板本体~~（背景画像`background_kitchen.png`に統合したため不要になりました） | - | - | - |

※以前の案にあった `okonomiyaki_base.png` は `body_00_raw.png` に統合しました。

**重要：5枚は同じ構図・同じ大きさで作る**
ゲーム中はこの5枚を同じ位置でパッと入れ替えるだけなので、**角度・中心位置・全体の大きさを揃えて生成する**のがポイントです。Geminiに1枚ずつ別々に頼むとズレやすいので、「同じ構図で、焼き色と面（表/裏）だけ変えたバリエーションを5枚」という形で、できれば1つの会話の中で連続して生成する（前の画像を参照させながら作る）のがおすすめです。ズレてしまった場合は、後でFigma等で位置合わせが必要になります。

## まとめ：優先着手順のおすすめ

1. **★★★のみ、まず一通り揃える**（本体系・鉄板・へら・トッピングアイコン4つ・焦げ跡パーツの「かんぺき」「こげた」各1〜2枚）→ これだけでゲームの見た目が一気に良くなります
2. 動かしてみて雰囲気を確認
3. ★★を追加（湯気、そこそこ／なま焼けパーツ、あおのり・かつおぶしの粒画像）
4. 余裕があれば★（星エフェクト、皿、フレームなど装飾要素）

## Geminiで生成する際の共通の呪文（プロンプトのベース案）

全画像で以下のようなベースプロンプトを使い回すと、絵柄がバラバラになりにくいです。

> 「かわいい手描き風のイラスト、絵本のような優しい線画タッチ、明るい暖色系の配色、背景は透過（白背景で生成後に透過処理する前提でも可）、子供向けアプリのアセット」

これに「お好み焼きの生地、焼く前」「黒い焦げ跡の部分パーツ、円形の一部分だけ」のように、そのパーツ固有の指示を続ける形がおすすめです。
